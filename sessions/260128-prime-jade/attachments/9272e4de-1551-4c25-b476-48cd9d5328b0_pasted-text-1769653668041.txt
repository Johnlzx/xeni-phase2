User Story
User Story
As a Home Office caseworker reviewing applications for the government, I want documents to follow a clear, consistent naming convention so that I can immediately understand what a document is and who it relates to just by looking at the file name.

Detailed Description
User Need (Problem)
Immigration applications often contain many documents that look similar at a glance (e.g., multiple bank statements, payslips, letters). File names received directly from clients or third parties are inconsistent and often unhelpful.

Clear, standardised file names allow reviewers to quickly recognise documents without opening them, particularly when:

Reviewing applications
Downloading files or ZIP bundles
Preparing or checking submissions
Suggested Solution (Step-by-Step Behaviour)
1. Apply a Consistent Naming Convention
Xeni generates clear, structured document names using the standard pattern:

who_documentType_mostRecentDate

2. Clearly Identify Who the Document Relates To
The file name includes a short identifier indicating who the document relates to, such as:

applicant
sponsor
adviser
dependant (if applicable)
3. Clearly Identify the Document Type
The file name includes a recognisable document type, for example:

payslip
bankStatement
passport
letterOfConsent
employmentContract
proofOfCohabitation
proofOfAccommodation
4. Include a Date Only When Relevant
Time-based documents (e.g., bank statements, payslips):

Include the most recent relevant date in MMYY format
Example: 0126 for January 2026
Non time-based documents (e.g., passports, consent letters):

Omit the date portion entirely
5. Use the Same Convention Everywhere
The generated file name is used consistently across:

Document lists
Individual file downloads
ZIP exports
